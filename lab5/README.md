##  Цель работы
Реализовать программу на C++ для умножения квадратных матриц с использованием технологии MPI (Message Passing Interface), провести серию экспериментов на суперкомпьютере «Сергей Королёв» с различным количеством процессов, измерить время выполнения, производительность, ускорение и эффективность параллелизации.

## Описание реализации

### C++ модуль (`matrix_multi.cpp`)
- **Алгоритм:** Блочное умножение с распределением строк матрицы A между процессами
- **Параллелизация:** MPI_Scatterv/MPI_Gatherv для распределения A, MPI_Bcast для рассылки B 
- **Хранение данных:** Линейные массивы `std::vector<double>` для эффективной передачи 
- **Ввод/вывод:** Процесс 0 читает матрицы из файлов, записывает результат
- **Замер времени:** `MPI_Wtime()` с барьерной синхронизацией `MPI_Barrier`
- **Обработка ошибок:** Проверка открытия файлов, MPI_Abort при исключениях на процессе 0

### Python скрипт (`verif.py`)
Автоматизация бенчмарка и визуализация:
- Генерация пар случайных матриц с фиксированным seed для воспроизводимости
- Парсинг времени выполнения и GFLOPS из stdout (regex)
- Усреднение результатов с расчётом std/min/max
- Сохранение результатов в `results.csv` (графиков нет, т.к. matplotlib не поддерживается на python 3.6.8)

## Методика экспериментов

**Размеры матриц:** `200, 400, 800, 1200, 1600, 2000`

**Параметры генерации:**
- Тип: квадратные матрицы `n × n`
- Количество процессов: 1, 2, 4, 8
- Повторов на конфигурацию: 3
- Диапазон элементов: `[-5, 5]` (равномерное распределение)

### Скрипт startMPI.pbs
#!/bin/bash
#SBATCH --job-name=matrix_multi
#SBATCH --time=0:05:00
#SBATCH --nodes=1
#SBATCH --ntasks=8
#SBATCH --partition=batch
#SBATCH --output=job_%j.out
#SBATCH --error=job_%j.err

module load intel/mpi4
srun ./matrix_multi A.txt B.txt result.txt

## Анализ результатов
1) Почти идеальное ускорение
- На 2 процессах: 97-100% эффективности
- На 4 процессах: 91-100% эффективности
- На 8 процессах: 90-100% эффективности
2) Кубическая зависимость подтверждена
- При увеличении n в 2 раза время растёт в ~8 раз
- Пример: n=400 → 0.64s, n=800 → 4.75s (×7.4), n=1600 → 36.89s (×7.8)
3) Минимальные накладные расходы MPI
- Даже на малых матрицах (n=200) эффективность >97%
- Это говорит о правильной организации коммуникаций
4) Стабильность результатов
- Эффективность не падает с ростом размера матрицы
- На больших матрицах (n≥1200) сохраняется 93-96% на 8 процессах

## Производительность (GFLOPS)
Для n=2000 на 8 процессах:
- GFLOPS = (2 × 2000³) / (9.36 × 10⁹) = 16 / 9.36 ≈ 1.71 GFLOPS
Для n=2000 на 1 процессе:
- GFLOPS = (2 × 2000³) / (71.7 × 10⁹) = 16 / 71.7 ≈ 0.22 GFLOPS
Ускорение производительности: ×7.8
